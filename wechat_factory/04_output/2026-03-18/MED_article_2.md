# 静态榜单≠安全：DAS 动态红队让 94% “高分”答案当场翻车

## 研究概览
- TUM、牛津等机构提出 DAS（Dynamic, Automatic & Systematic）红队框架，针对医疗 LLM 的鲁棒性、隐私、公平与幻觉四大维度进行“活体”压力测试（arXiv:2508.00923v2，2026.03.09）。
- 系统内置多名对抗代理，可自行改写病历、嵌入偏见、引导违规提问，直到模型彻底暴露隐患。

## 核心发现
1. **Benchmarking Gap**：15 个主流模型在 MedQA 等静态榜单平均 >80%；但在 DAS 的动态鲁棒性测试中，**94% 原本答对的题目被轻微扰动就答错**。
2. **HealthBench 也失守**：面对开放式健康问答数据，顶级模型仍有 >70% 的失败率，说明“刷榜”并不能转化为真实可靠性。
3. **隐私脆弱**：在模拟 HIPAA/GDPR 情境下，**86% 的测试都能诱发隐私泄露**（如透露病人身份、无授权下分享病历）。
4. **公平性偏移**：通过“认知偏差 + 社会属性”组合提示，可让模型在 81% 情况下改变临床推荐，暴露潜在歧视风险。
5. **医学幻觉**：构建七类医学特有幻觉标签后发现，**74% 的攻击情境**会触发凭空指南、编造药物禁忌等严重错误。

## DAS 框架如何运作
- **多代理协同**：场景生成 Agent 不断变换病历细节；攻击 Agent 迭代强化 jailbreak；审计 Agent 校准输出是否违反安全准则。
- **临床对齐验证**：红队生成的案例与持证医生有高一致性，避免“太学术/不现实”的对抗。
- **四大轴线联动**：同一模型会经历数据扰动、隐私诱导、偏见注入、事实核查等多轮回合，直到通过或被判定失败。

## 业务意义
1. **监管准备**：FDA 与欧盟都在强调“动态审计”，DAS 提供一个可落地的自动化工具链，帮助企业提前发现合规风险。
2. **模型选型**：DAS 在 15 个模型间发现显著的排名反转——某些 MedQA 冠军在动态测试中完败，提示我们选型时不能只看榜单。
3. **安全基线**：在投产前，应先跑过 DAS 类红队脚本，把隐私/公平/幻觉等指标做成常规健康度报告。

## 我们的建议
- **建立“活体”评测管线**：把传统静态测试的结果当作起点，再接入 DAS 式持续红队，让模型与攻击者“共同进化”。
- **反馈闭环**：将红队暴露的失败样本回流训练，用作安全微调或流程守卫（例如加上权限判断、可解释路径）。
- **跨部门联动**：安全、合规、临床专家应共同定义“不可接受的失败模式”，再由红队展开针对性攻击。

DAS 把“医疗大模型是否可信”从单一分数变成一个全天候监控任务。想靠一次考试证明模型靠谱的时代结束了。
