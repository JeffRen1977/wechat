# 微信排版 Markdown 规范

<!-- 定义正文结构、标题层级、引用与列表样式，便于转成公众号 HTML；与 DESIGN、PUBLISHING 一致 -->

## 角色定位（给 Agent）

- 以**拥有稳定阅读量经验的公众号主编**视角写作：**先共情、再讲清楚、最后能带走**。
- **开头**：尽量用**一个小故事、社会热点、或一句扎心设问**切入，避免「本文介绍……」式冷启动。
- **行动收尾**：文末必须有 **3 条可立即执行的建议（Action Items）**，用编号列表列出，动词开头、每条一行说清楚。

## 语气与读者（公众号向）

- **生动有趣**：开头钩子 + 类比/口语过渡；避免整篇「说明书腔」。
- **浅显易懂**：默认读者为**非专业大众**；专业词首次出现用**一句话人话**解释；长句拆短；复杂逻辑用「先结论、再展开」。
- **热点/视频稿**：若素材来自 YouTube 或热点，H2 可更活泼（如「这条视频在讲啥」「对我们有啥用」「要注意啥」），但仍保持**事实准确**，并在知识库或文末保留**原视频/来源链接**便于追溯。

## 内容逻辑框架（三垂类通用）

### 黄金圈（Why → How → What）

- 不要开篇就定义「这是什么产品/论文」；先用 **Why**：**为什么现在值得关注**（趋势、痛点、一条新闻）。
- 再 **How**：机制或做法用**人话 + 类比**。
- 最后 **What**：读者具体能做什么、要注意什么边界。

### SCQA（情境-冲突-问题-答案）

| 领域 | S+C（情境+冲突） | Q（问题） | A（答案） |
|------|------------------|-----------|-----------|
| **教育** | 学习场景里的真实困境、时间不够、信息过载 | 读者心里会问「那我怎么办？」 | AI 或方法如何**减负、提效**，给可执行步骤 |
| **健康** | 健康焦虑、信息鱼龙混杂、某条热点说法 | 「靠谱吗？适合我吗？」 | **科普级**结论 + 何时该问医生（不替代诊疗） |
| **财经** | 市场波动、政策/产品热点 | 「普通人怎么理解？怎么防坑？」 | **风险意识** + 配置/决策框架（非个股推荐话术） |

## 结构

- **单一级 H1**：整篇文章仅一个一级标题，作为主标题，与公众号标题一致或略扩展。
- **小节**：用 **H2** 划分大段，**H3** 作子节（可选）。可穿插 **Why / How / What** 或 **SCQA** 节奏，不必显式写小标题「SCQA」。
- **字数**：单篇 **1500–2000 字**；科技向可对标《机器之心》的信息密度，但**语气以通俗、好读为先**。

## 段落与「视觉化」排版（公众号审美）

- **段落长度**：**严禁大段文字**。在手机上以**每段不超过 3 行**为参考；超过就拆段，段间空一行。
- **加粗**：**核心观点、结论句、数字、对比结论**必须 **加粗**；不宜整段加粗。
- **引用块 `>`**：用于**金句、核心结论、典型案例摘录**单独成块，便于转 HTML 后形成强调区。
- **分割线**：大段之间可用单独一行的 `---`（Markdown 水平线）或 `***` 区分节奏（不要滥用，每篇 1～3 处即可）。
- **列表**：
  - 并列要点用 `-` 或 `1.`；**每条列表项必须紧跟正文**（同一行或下一行紧接文字），不要只写 `-` 或 `1.` 后空行再写内容，否则转公众号后容易出现「只有圆点没有字」。
  - 适当用 **emoji 作列表前缀** 增加层次（如 💡 提示、✅ 要点、⚠️ 风险），与正文语气一致即可，不必每行都加。
- **代码/公式**：若出现算法名、模型名、简短公式，可用行内 `code`；长公式可改为文字描述或配图说明。
- **专有名词**：首次出现可简要解释，或与 `02_knowledge_base` 中术语保持一致。

## 标题（爆款取向）

- 正式 **H1** 仍须 **≤32 字**（公众号限制）。
- 撰稿前阅读 `viral_titles.txt`；在草稿**开头**（H1 之前）可用 HTML 注释列出 **5 个备选标题**，再选定最佳作为唯一 H1，例如：
  - `<!-- 备选标题：1.悬念型 … 2.收益型 … 3.对比型 … 4.警示型 … 5.好奇型 …选用：第X个 -->`
- 五类风格提示：**悬念**（到底……）、**收益**（读完你能……）、**对比**（A和 B 差在哪）、**警示**（别忽视……）、**好奇**（为什么最近都在聊……）。

## 配图

- **封面图**：每篇对应一张封面，路径为 `wechat_factory/05_assets/images/` 下文件，建议命名含日期与领域，如 `YYYY-MM-DD_MED_cover.png`。
- **正文配图**：文中若引用图，路径同样指向 `05_assets/images/`；转 HTML 时由发布脚本上传并替换为公众号图片 URL。脚本会结合 **H2 段落大意** 生成配图，正文请保证 **H2 下先有实质内容**再插图占位。
- **图注**：在 Markdown 中用图片下方简短说明即可，与 DESIGN 中「封面与配图」约定一致。

## 统一结尾引导（CTA）

- 在 **Action Items** 之后、**来源链接**之前，增加一小段固定结构的互动话术（可微调用词，结构保持一致）：
  - 一句总结 + 「如果这篇对你有用，欢迎 **点赞、在看**；想持续收到类似解读，点个 **关注** 不迷路。」
  - 避免违规诱导分享话术，以平台规则为准。

## 主编自检（发布前心理检查，非单独 Agent）

- [ ] 开头 3 段内是否有**情绪或场景**，而非纯定义？
- [ ] 是否显式走过 **Why**（或 SCQA 的痛点）？
- [ ] 是否有 **3 条 Action Items**？
- [ ] 是否无「手机上一坨字」段落（段均不过长）？
- [ ] 金句/结论是否至少一处用了 **引用块**或 **加粗 + 单独成段**？
- [ ] 标题是否 ≤32 字且来自备选中的较优解？

## 与公众号的衔接

- 标题需在 **32 字以内**（公众号单篇图文限制）。
- 正文 HTML 由 `scripts/md2wechat.sh` 等从本 Markdown 生成，再经上传素材、提交草稿；详见 PUBLISHING.md。
