# AI 论文解析公众号自动化系统 — 设计文档

## 1. 项目概述

### 1.1 目标

打造一个能够**每天自动产出 5 篇高质量 AI 论文解析**的公众号自动化系统，形成「感知 → 处理 → 输出」的闭环，在 OpenClaw 2026 体系下以**多智能体协同 (Multi-Agent Workflow)** 方式运行。

### 1.2 核心指标

| 指标 | 目标 |
|------|------|
| 日产出 | 5 篇 |
| 质量 | 对标《机器之心》等头部科技媒体 |
| 字数 | 单篇 1500–2000 字 |
| 领域 | 医疗 AI / 金融 AI / 教育 AI（可扩展） |

### 1.3 设计原则

- **多智能体协同**：各环节由专用 Agent/Skill 负责，而非单一大模型流水线。
- **可观测、可回滚**：Workspace 结构化存储 + Git 版本控制，防止误删与便于审计。
- **成本与质量平衡**：本地模型（Ollama）做初筛/翻译/去重，云端大模型（Claude）做深度解读与润色。

## 2. 高层系统架构

系统分为**四层**：数据源层、处理引擎层、内容仓库层、发布辅助层。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        发布辅助层 (Publishing)                            │
│  Markdown→HTML / 微信公众号排版 / 定时发布 / 封面图                        │
└─────────────────────────────────────────────────────────────────────────┘
                                      ▲
┌─────────────────────────────────────────────────────────────────────────┐
│                        内容仓库层 (Workspace)                             │
│  04_output/ 按日期归档 · 05_assets/ 封面与配图 · Git 版本控制              │
└─────────────────────────────────────────────────────────────────────────┘
                                      ▲
┌─────────────────────────────────────────────────────────────────────────┐
│                        处理引擎层 (Multi-Agent)                           │
│  感知 → 认知 → 创作 → 存储（Claude / 本地模型 / MCP Skills）               │
└─────────────────────────────────────────────────────────────────────────┘
                                      ▲
┌─────────────────────────────────────────────────────────────────────────┐
│                        数据源层 (Sources)                                 │
│  arXiv / PubMed / Nature / Medium / 搜索引擎 / 网页快照                    │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.1 核心流程（四阶段）

| 阶段 | 名称 | 职责 | 主要能力 |
|------|------|------|----------|
| 1 | **感知 (Perception)** | 发现最新高质量论文与动态 | 定时触发；搜索 + 浏览器工具扫描全球顶尖 AI 动态 |
| 2 | **认知 (Cognition)** | 深度理解论文内容 | Claude Opus/3.5 对 PDF 深度解码，提取核心逻辑、图表与结论 |
| 3 | **创作 (Creation)** | 生成可发布文案 | 基于「爆款模板」进行标题与正文重构，风格与字数可控 |
| 4 | **存储 (Persistence)** | 结构化落盘与版本管理 | 内容写入 Workspace 指定目录，配合 Git 防误删、可回滚 |

## 3. 文件架构设计 (File Structure)

建议在 `~/.openclaw/workspace/wechat_factory/` 或项目内等价路径下建立以下目录（与代码仓库可分离或映射）：

```
wechat_factory/
├── 01_sources/                 # 原始素材区
│   ├── papers_pdf/             # 下载的论文原文 (PDF)
│   └── web_snapshots/          # 网页抓取的原始文本/HTML
├── 02_knowledge_base/          # 长期知识库
│   ├── medical_ai.md           # 医疗 AI 专有名词与背景知识
│   ├── finance_ai.md           # 金融 AI 行业趋势
│   └── edu_ai.md               # 教育 AI 政策与技术
├── 03_templates/               # 爆款模板区
│   ├── viral_titles.txt        # 标题句式库（高点击）
│   └── article_style.md        # 微信排版 Markdown 规范
├── 04_output/                  # 最终产出区
│   └── YYYY-MM-DD/             # 按日期分类的文章
│       ├── MED_article.md      # 医疗 AI 篇
│       ├── FIN_article.md      # 金融 AI 篇
│       └── EDU_article.md      # 教育 AI 篇（示例命名）
└── 05_assets/                  # 媒体素材
    └── images/                 # AI 生成的封面图/配图
```

### 3.1 目录职责说明

| 目录 | 用途 | 读写方 |
|------|------|--------|
| `01_sources` | 原始论文 PDF、网页快照，只增不删（除策略性清理） | 感知层写入；认知层读取 |
| `02_knowledge_base` | 领域术语、政策、趋势，供生成时引用 | 人工维护 + Agent 可读 |
| `03_templates` | 标题句式、正文风格、排版规范 | 人工维护 + 创作层读取 |
| `04_output` | 按日期存放当日 5 篇成文 Markdown | 创作层写入；发布层读取 |
| `05_assets` | 封面图、配图路径记录 | 图像生成 Skill 写入；排版/发布引用 |

## 4. 必需的 Skills（能力扩展 / MCP）

在 OpenClaw 中需配置以下 MCP (Model Context Protocol) 技能插件：

| Skill 名称 | 推荐工具 | 具体用途 |
|------------|----------|----------|
| **Deep Search** | google-search / brave-search | 突破 LLM 知识截止日期，获取 2026 年最新论文链接与摘要 |
| **Web Browser** | puppeteer / playwright | 模拟访问 arXiv、PubMed、Medium 等，抓取全文与元数据 |
| **PDF Parser** | sequential-strategy | 深度阅读 30+ 页学术论文，提取章节、图表、结论，不漏关键信息 |
| **Filesystem** | local-files | 自动创建目录、重命名文件、整理 01_sources 与 04_output |
| **Image Gen** | dalle-node / flux-mcp | 根据文章摘要生成科技感封面图，输出到 05_assets/images |

## 5. 需配合使用的外部工具

除 OpenClaw 核心外，完成「最后一公里」发布与稳定性所需工具：

| 工具 | 用途 |
|------|------|
| **Ollama (本地)** | 使用 Qwen 2.5 / Llama 3.3 做初步翻译、去重与初筛，降低 API 成本后再交给 Claude 润色 |
| **Pandoc / Markdown-to-WeChat** | 将 Markdown 转为带样式的 HTML，适配微信公众号排版 |
| **Cron (系统级)** | 定时触发流水线；OpenClaw 异常退出时由 Cron 重启进程，保证日更 |
| **Git / GitHub** | 对 Workspace（尤其是 01_sources、04_output、02_knowledge_base）做版本控制，防止误删与便于回溯 |

## 6. 「自动化总编」指令模板 (Action Template)

可将以下内容作为 `~/.openclaw/agents/dev/agent/INSTRUCTIONS.md` 的核心段落，用于驱动每日自动化流程。

### 6.1 任务定位

- **公众号定位**：如「AI 行业前哨」。
- **领域**：医疗 AI、金融 AI、教育 AI（可配置扩展）。

### 6.2 每日动作清单

1. **扫描**  
   使用 Deep Search（如 Google Search）查找过去 24 小时内 **[医疗/金融/教育]** 相关高引或高影响力论文与报道。

2. **研读**  
   下载原文至 `01_sources/papers_pdf/` 或抓取至 `01_sources/web_snapshots/`；用 PDF Parser / 浏览器工具提取关键结论，并与 `02_knowledge_base` 中历史进展对比。

3. **拟题**  
   基于 `03_templates/viral_titles.txt` 生成 5 个具有点击欲望的标题，择优 1 个作为当日该领域文章标题。

4. **撰稿**  
   风格对标《机器之心》，字数 1500–2000 字，Markdown 格式，遵循 `03_templates/article_style.md`。

5. **归档**  
   将成文写入 `workspace/wechat_factory/04_output/YYYY-MM-DD/`，并按领域命名（如 `MED_article.md`）。

6. **配图（可选）**  
   调用 Image Gen 根据摘要生成封面，保存至 `05_assets/images/` 并在文章 Frontmatter 或文末注明路径。

## 7. 领域与质量策略（示例：医疗 AI）

为避开低质论文，可对「医疗 AI」单独做深度搜索与过滤策略，例如：

- **来源优先级**：  
  《柳叶刀》(The Lancet)、《Nature Medicine》、PubMed 高引、顶会（如 NeurIPS/ICML 医疗轨道）等。
- **过滤规则**：  
  预定义关键词 + 引用量/期刊影响因子阈值（可由配置文件或 Knowledge Base 维护）。
- **去重**：  
  与 `02_knowledge_base/medical_ai.md` 中已解析论文列表比对，避免重复选题。

后续可单独撰写「医疗 AI 深度搜索 Prompt」与「金融/教育 AI」的同类策略文档。

## 8. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 单日论文数量不足或质量差 | 多数据源 + 多关键词；放宽时间窗口（如 48h）；人工备选池 |
| PDF 解析漏掉图表 | 使用 sequential-strategy 分段阅读；关键图表单独用 Image Gen 或截图补全 |
| 公众号 API/排版限制 | 使用 Markdown-to-WeChat 或 Pandoc 生成合规 HTML；预留人工校对步骤 |
| OpenClaw 进程崩溃 | 系统 Cron 定时重启；关键目录 Git 每日提交 |
